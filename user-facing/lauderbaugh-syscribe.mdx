\# Real-Time Activity Updates & Draft Regeneration ## What's New This release introduces live, real-time updates across the dashboard and drafts review pages. Instead of needing to manually refresh to see the latest status, the app now automatically reflects changes as they happen — including new drafts being created, drafts being updated, deployments changing status, and commits finishing processing. You'll also find a new \*\*Regenerate\*\* button on pull request review pages, letting you kick off a fresh round of draft generation without leaving the page. --- ## Real-Time Updates ### What changed The dashboard and draft review pages now listen for live events in the background. When something changes — for example, a commit finishes processing or a draft moves to a new status — the page updates automatically. ### What you'll see - \*\*Dashboard activity feed\*\* — The activity list refreshes when new events arrive. A subtle indicator appears when there is new activity since you last looked. - \*\*Draft review page\*\* — If a draft you're reviewing is updated while you have the page open, the view reflects the latest state without you needing to reload. ### What to expect - The connection to the live event stream is established automatically when you open a page. If the connection drops, the app retries with increasing delays (between 1 second and 30 seconds) until it reconnects. - Heartbeat checks run every 30 seconds to keep the connection healthy. - If your session token expires, the connection will attempt to refresh it automatically. --- ## Regenerate Drafts ### What changed A \*\*Regenerate\*\* button now appears on pull request review pages. ### When you can use it You can regenerate drafts when: - No drafts have been generated yet for the pull request, \*\*or\*\* - At least one draft exists but regeneration is permitted based on the current state. The button is unavailable (greyed out) when regeneration is not applicable — for example, when drafts are currently being generated. ### How to use it 1. Open a pull request from the \*\*Drafts\*\* page. 2. If regeneration is available, you'll see a \*\*Regenerate\*\* button near the top of the review panel. 3. Select \*\*Regenerate\*\*. The page will show a loading state while the new drafts are being prepared. 4. Once complete, the draft tabs update automatically with the fresh content — no manual refresh needed. --- ## Activity Status on the Dashboard ### What changed The activity feed on the dashboard now shows a richer set of statuses that reflect the full lifecycle of a commit — from when it is first received, through draft generation, all the way to deployment. Each status has a clear label and visual indicator so you can tell at a glance where things stand. --- ## Limitations - Real-time updates require an active internet connection. If you're offline, the page will not update until connectivity is restored and the connection re-establishes. - The Regenerate button is only available on the draft review page — you cannot trigger regeneration from the dashboard or the drafts list.
