\# Documentation Sources Streamlined ## What changed This release removes built-in support for automatically syncing documentation from \*\*GitBook\*\*, \*\*GitHub Pages\*\*, and \*\*Notion\*\*. Previously, the system could pull content directly from these third-party platforms and convert it into a unified format. That automatic syncing capability has been removed. The document type system has also been updated — the recognized document types are now limited to \*\*Internal\*\*, \*\*User-Facing\*\*, and \*\*API\*\*. ## What this means for you If you were using the automatic sync feature to pull documentation from GitBook, GitHub Pages, or Notion, \*\*that functionality is no longer available\*\*. Content will no longer be fetched or converted from those sources automatically. ### Document types Going forward, when categorizing or tagging documentation, you'll work with three document types: | Type | Description | |------|-------------| | \*\*Internal\*\* | Documentation intended for internal use only | | \*\*User-Facing\*\* | Documentation written for end users | | \*\*API\*\* | Documentation describing API usage | Attempting to use a document type outside of these three will no longer be accepted. ## What to do next If your workflow relied on syncing content from GitBook, GitHub Pages, or Notion, you will need to manually import or manage that content going forward. Check with your administrator or the development team for guidance on alternative approaches for bringing external documentation into the system.
