\## Documentation History: New Audit Trail for Your Documentation Activity ### What's new You can now view a complete history of every documentation event across your organization — all in one place. This release introduces a \*\*Documentation History\*\* log that tracks what happened to each doc draft, who did it, and when. The history log captures events including: - \*\*Draft generated\*\* — the system produced a new documentation draft from a code change - \*\*Draft not applicable\*\* — the system determined a change didn't require a doc update - \*\*Agent interaction\*\* — you or a teammate asked the AI assistant to revise a draft - \*\*Manual edit\*\* — a team member directly edited draft content - \*\*Approval granted\*\* — a draft was approved and marked ready to publish - \*\*Approval rejected\*\* — a draft was rejected, with a reason recorded - \*\*Deployment succeeded / failed\*\* — the doc was published (or attempted) to its destination --- ### How to use it \*\*On a PR review page:\*\* 1. Open any pull request's documentation review. 2. You'll see a new \*\*History\*\* tab alongside the draft tabs (Internal, User-Facing, API). 3. Select the \*\*History\*\* tab to see a timeline of all documentation events tied to that pull request. \*\*On the organization-wide history page:\*\* 1. Navigate to your organization's settings or activity area. 2. Open the \*\*Documentation History\*\* page. 3. Use the \*\*event type filter\*\* to narrow results to a specific activity (e.g. approvals only, deployments only). 4. Use the \*\*date range selector\*\* to show events from the last 7 days, last 30 days, or all time. 5. Results are paginated — use the page controls to browse older entries. --- ### What to expect - Every doc-related action — generation, editing, approval, deployment — is now recorded automatically. You don't need to do anything to enable this. - The history log is \*\*read-only\*\*. You can view and filter entries, but not delete or edit them. - Entries include a short summary of what happened, a badge indicating the event type, and a timestamp. - If a draft was generated with an AI model, the history entry records which model was used and the token usage for that generation. - On the PR review page, the History tab shows events scoped to that specific pull request. The organization-wide page shows all events across all repositories. - Events tied to bulk approvals (approving all drafts at once) appear as a single history entry listing all approved doc types.
